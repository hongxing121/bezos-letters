# Mechanical Turk

## 基本信息
- **全称**：Amazon Mechanical Turk (MTurk)
- **类型**：自建业务
- **首次提及**：[[2006 贝佐斯致股东信]]（作为 AWS 十种网络服务之一间接提及）
- **提及信件数**：0 封（在 1997-2020 年致股东信中未被直接提及）

## 发展历程

Mechanical Turk 于 2005 年 11 月公开发布，是亚马逊推出的众包平台，允许企业将需要人类智能完成的微任务（如图像标注、数据验证、调查问卷等）分发给全球工人。该平台名称源自 18 世纪"会下棋的机器人"骗局——看似机器自动完成的任务，实际由人类在幕后操作。

- **2005-2006年**：MTurk 作为 AWS 早期服务之一推出。2006 年信中提到 AWS 已有"十种不同的网络服务"，MTurk 是其中之一。贝佐斯描述 AWS 团队"瞄准的是开发者普遍面临的基本需求"，MTurk 属于这一愿景下将人类智能包装为 API 服务的创新尝试。（[[2006 贝佐斯致股东信]]）

**说明**：Mechanical Turk 在贝佐斯致股东信中从未被单独提及或详细讨论。它是 AWS 早期服务组合的一部分，但在信件中的关注度远低于 S3、EC2 等核心服务。MTurk 的重要性更多体现在其概念上的先驱意义——将"人类智能"封装为可调用的网络服务（Human Intelligence Task, HIT），是亚马逊"万物皆服务"理念的早期体现。

## 贝佐斯评价

> "通过 AWS，我们正在围绕一个全新的客户群体——软件开发者——构建一项新业务。目前我们已经提供了十种不同的网络服务。" —— [[2006 贝佐斯致股东信]]（MTurk 为十种服务之一）

## 体现的核心概念
- [[自助服务平台]] — 将人类智能封装为自助式 API，无需守门人
- [[发明与失败]] — 将人类劳动以编程接口方式提供，概念上极具创新性
- [[客户至上]] — 为开发者解决机器无法完成但人类擅长的任务需求
